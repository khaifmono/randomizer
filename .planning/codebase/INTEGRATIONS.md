# External Integrations

**Analysis Date:** 2026-03-26

## APIs & External Services

**API Documentation:**
- Scalar API Reference - Interactive OpenAPI documentation UI served at `/reference`
  - SDK/Client: @scalar/hono-api-reference 0.9.24
  - Configuration: `apps/api/src/lib/configure-open-api.ts`
  - Environment-gated: Only available in non-production environments

## Data Storage

**Databases:**
- Cloudflare D1 (SQLite)
  - Connection: Wrangler binding `DB` configured in `apps/api/wrangler.jsonc`
  - ORM: Drizzle ORM 0.44.7
  - Schema location: `apps/api/src/db/schema/`
  - Migrations: `apps/api/src/db/migrations/`
  - Database name: `base-project-api`
  - Database ID: `00000000-0000-0000-0000-000000000000` (placeholder)

**Tables:**
- `users` - User account storage with fields:
  - `id` (text, primary key, UUID generated)
  - `username` (text, unique)
  - `email` (text, unique)
  - `password_hash` (text)
  - `created_at` (integer, timestamp)
  - `updated_at` (integer, timestamp)

**File Storage:**
- Cloudflare Workers ASSETS binding - Static file serving
  - Binding name: `ASSETS`
  - Used for: Frontend SPA assets (HTML, JS, CSS)
  - Configuration: `apps/api/src/lib/create-app.ts`
  - Fallback: Serves `index.html` for SPA routing

**Caching:**
- TanStack React Query - Client-side data caching
  - Default stale time: 1 minute
  - Refetch on window focus: Disabled
  - Configuration: `apps/web/src/lib/query-client.ts`

## Authentication & Identity

**Auth Provider:**
- Custom implementation
  - Approach: Password hash storage in database
  - Secret: Configurable via `AUTH_SECRET` environment binding
  - User table: `apps/api/src/db/schema/user.ts`
  - Note: Only password hash stored, never plaintext password (see schema comments at `apps/api/src/db/schema/user.ts`)

## Monitoring & Observability

**Logging:**
- Pino structured logging framework
  - Integration: hono-pino middleware
  - Configuration file: `apps/api/src/middlewares/pino-logger.ts`
  - Level: Configurable via `LOG_LEVEL` environment binding (default: "info")
  - Request ID tracking: From Cloudflare headers (`cf-ray`, `x-request-id`) or generated UUID
  - Browser-safe output: Configured to not use Node.js transports in Workers environment

**Error Tracking:**
- Built-in error handling via Stoker
  - Error handler: `stoker/middlewares.onError`
  - 404 handler: `stoker/middlewares.notFound`
  - Implementation: `apps/api/src/lib/create-app.ts`

## CI/CD & Deployment

**Hosting:**
- Cloudflare Workers - Serverless edge compute platform
  - Main entry: `apps/api/src/app.ts`
  - Compatibility date: 2024-11-05
  - Compatibility flags: nodejs_compat
  - Configuration: `apps/api/wrangler.jsonc`

**Frontend Hosting:**
- Cloudflare Workers ASSETS binding
  - Served via Workers fetch handler in `apps/api/src/lib/create-app.ts`
  - SPA fallback to index.html for client-side routing

**CLI:**
- Wrangler 4.54.0 - Official Cloudflare Workers CLI
  - Dev command: `wrangler dev`
  - Deploy command: `wrangler deploy --minify`
  - Database migration: `wrangler d1 migrations apply base-project-api`

## Environment Configuration

**Required env bindings (Cloudflare Workers):**
- `DB` - D1 database instance (type: D1Database)
- `ASSETS` - Static asset fetcher (type: Fetcher)
- `AUTH_SECRET` - Authentication secret (type: string)
- `LOG_LEVEL` - Logging level (type: string, default: "info")
- `NODE_ENV` - Environment mode (type: string)

**Secrets location:**
- Managed via Cloudflare wrangler.jsonc and Cloudflare dashboard
- See: `apps/api/wrangler.jsonc` for binding definitions

## API Routes & Endpoints

**Base Path:** `/api` (defined in `apps/api/src/lib/constants.ts`)

**Routes:**
- `GET /api/doc` - OpenAPI specification document (non-production only)
- `GET /api/reference` - Interactive API documentation via Scalar (non-production only)
- `GET /api/health` or similar - Health check (generated by stoker)
- `/api/users` - User management routes
  - Implementation: `apps/api/src/routes/user/`
  - Handlers: `apps/api/src/routes/user/user.handlers.ts`
  - Validation: `apps/api/src/routes/user/user.routes.ts`

## API Client

**Type-safe RPC Client:**
- Package: `@base-project/api-client`
- Location: `packages/api-client/src/index.ts`
- Implementation: Hono RPC client (`hc` from hono/client)
- Type inference: Auto-generated from `@base-project/api` router type
- Consumer: Web app uses via import `@base-project/api-client`

**Error Schema:**
```typescript
{
  error: {
    issues: Array<{
      code: string;
      path: (string | number)[];
      message?: string;
    }>;
    name: string;
  };
  success: boolean;
}
```

## Frontend-Backend Communication

**Dev Proxy:**
- Vite proxy configuration in `apps/web/vite.config.ts`
- `/api` requests proxied to `http://localhost:8787`
- Allows local development with backend running on Wrangler dev server

**Query Integration:**
- React Query manages all API communication
- Built-in retry and stale-while-revalidate patterns
- Devtools available via `@tanstack/react-query-devtools`
