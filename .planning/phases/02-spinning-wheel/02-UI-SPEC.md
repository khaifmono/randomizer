## Spacing Scale

Declared values (multiples of 4 only). Tailwind v4 — use spacing utility classes that map to these values.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label-to-icon gap on buttons, gap between remove (×) button and item text |
| sm | 8px | Compact element spacing — padding inside item list rows, gap between quick-add input and Add button |
| md | 16px | Default element spacing — card padding, section internal spacing, gap between form elements |
| lg | 24px | Section padding — gap between wheel canvas area and item management panel |
| xl | 32px | Layout gaps — gap between the two main columns (wheel + panel) on desktop |
| 2xl | 48px | Major section breaks — top/bottom padding of the WheelTab container |
| 3xl | 64px | Page-level spacing — not applicable to this phase (page shell from Phase 1) |

**Exceptions:**
- Spin button minimum touch target: 44px height (accessibility; overrides standard 40px button height)
- Wheel canvas: fills available width up to 480px, maintains 1:1 aspect ratio — not a spacing token, a layout constraint

## Typography

All text uses `font-sans` (Public Sans, variable). Two weights only.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Item list entries, textarea placeholder, helper text, history entries |
| Label | 14px | 600 (semibold) | 1.4 | Input labels, section headings within the panel ("Items", "Add Item"), button text |
| Heading | 20px | 600 (semibold) | 1.2 | Winner overlay name — the announced winner displayed bold over the wheel center |
| Display | 28px | 600 (semibold) | 1.1 | Not used in this phase |

**Notes:**
- Winner overlay uses Heading (20px / 600) — large enough to be clearly readable at canvas size, not so large it breaks on long item names
- Body and Label share the same size (14px) — differentiated by weight only, which is intentional for compact panel layout
- No italic usage in this phase

4. Item count badge background (when shown — Phase 5, but token reserved now)

**Segment color palette (Canvas — Claude's discretion, D-04):**
Use a 10-color vibrantly-distinct palette cycling by index. Colors are drawn directly on Canvas — they do not use CSS variables. Palette:

| Index (mod 10) | Canvas fill color | Name |
|----------------|-------------------|------|
| 0 | `#6366f1` | Indigo |
| 1 | `#f59e0b` | Amber |
| 2 | `#10b981` | Emerald |
| 3 | `#ef4444` | Red |
| 4 | `#8b5cf6` | Violet |
| 5 | `#3b82f6` | Blue |
| 6 | `#f97316` | Orange |
| 7 | `#14b8a6` | Teal |
| 8 | `#ec4899` | Pink |
| 9 | `#84cc16` | Lime |

Adjacent segments always differ (cycling guarantees no two adjacent segments share a color for up to 10 items; for 10+ items the cycle repeats but adjacent contrast is maintained by the palette ordering — no two neighbors are the same hue).

Canvas text on segments: white (`#ffffff`) with a 1px black text shadow for legibility on all colors.

**Pointer triangle (D-03):** white fill, 2px black stroke, positioned at 12 o'clock outside the wheel radius.


- Canvas fills full container width (max 480px centered)
- Panel stacks below controls with 24px (lg) gap

## Interaction Contract

### Spin Interaction (D-05, D-06, D-07, D-08)

| State | Visual |
|-------|--------|
| Idle — items present | Spin button enabled (accent background), wheel clickable |
| Idle — 0 items | Spin button disabled (muted, cursor-not-allowed), wheel click does nothing |
| Spinning | Spin button disabled + shows spinner icon (Lucide `Loader2` rotating); wheel canvas animates at 60fps |
| Winner overlay | Center overlay fades in over wheel; winner name in Heading (20px/600); semi-transparent accent tint panel; auto-fades after 2 seconds |
| Post-fade | Winner removed from list; wheel re-draws with updated segments; Spin button re-enabled |

**Animation spec (D-06):**
- Duration: 4–5 seconds total (randomized per spin within range)
- Rotations: 4–6 full 360° rotations before stopping (randomized per spin)
- Easing: `motion@^12` `animate()` with easing `[0.12, 0, 0.39, 0]` cubic-bezier (fast start, dramatic slowdown)
- Winner: pre-determined before animation starts (RESEARCH.md Pattern 1 — mandatory)

**Overlay animation spec (D-07):**
- Appear: `tw-animate-css` `animate-in fade-in zoom-in-95` — duration 200ms
- Dismiss: `tw-animate-css` `animate-out fade-out zoom-out-95` — duration 200ms
- Auto-dismiss after: 2000ms display + 200ms fade-out = 2200ms total before item removal fires

### Item Entry (D-01, D-02)

| Interaction | Behavior |
|-------------|----------|
| Type in quick-add input + press Enter or click Add | Appends item to list; clears input; focuses input again |
| Click "Bulk add" toggle | Expands a textarea below the quick-add row; collapses with second click |
| Type in bulk textarea + click "Add all" | Splits on newlines, trims each line, filters empty strings, appends all new items to list |
| Click × on item row | Immediately removes that item from list; no confirmation required |
| Click "Reset" | Restores all items to the snapshot taken at first page load (from localStorage); no confirmation required |

**Input validation:**
- Empty string: Add button is disabled when input is blank (trimmed)
- Duplicate items: allowed (no deduplication — user controls the list)
- Item max length: 40 characters (enforced via `maxLength` attribute on input and textarea parsing)

### Wheel Click (D-05)
- Clicking anywhere on the `<canvas>` element triggers a spin (same as Spin button)
- During active spin: click on canvas is ignored
- Cursor on canvas: `cursor-pointer` when idle, `cursor-not-allowed` when spinning

## States Reference

All component states the executor must implement:

### WheelCanvas
- `idle-empty` — renders placeholder circle with "Add items to spin" text centered, muted color
- `idle-ready` — renders full colored segments with item text
- `spinning` — animation running, pointer fixed, segments rotate
- `winner-overlay` — overlay rendered over canvas center with winner name

### WheelItemList
- `has-items` — scrollable list, each row: item text (body 14px/400) + × button (muted-foreground, hover destructive)
- `empty` — shows empty state (heading + body copy from Copywriting Contract above)
- `quick-add-only` — default view: input + Add button in a row
- `bulk-add-open` — input row + expanded textarea + Add all button

### Spin / Reset Controls
- `spin-enabled` — accent background, white text
- `spin-disabled` — muted background, muted-foreground text, Loader2 icon during spin
- `reset-visible` — text link style (not a filled button), muted-foreground color, shows only when items have been removed from original list
