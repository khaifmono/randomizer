## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (gap-1), inline padding |
| sm | 8px | Compact element spacing, card internal padding |
| md | 16px | Default element spacing (gap-4, p-4) |
| lg | 24px | Section padding (gap-6, p-6) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:

- Match card minimum width: 160px (fixed, flex-shrink-0) — prevents column layout jank during progressive reveal
- Match card connector line height: 2px — CSS border, not a spacing token
- Touch targets for Judge mode option rows: minimum 44px height — accessibility requirement for tappable pick targets
- Tab trigger padding follows the tab bar pattern: px-3 md:px-5 py-2 md:py-2.5

Source: established pattern from `randomizer.tsx` tab bar and `card-display.tsx` gap-4 pattern.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (text-base) | 400 (regular) | 1.5 |
| Label / muted counter | 14px (text-sm) | 400 (regular) | 1.4 |
| Heading | 20px (text-xl) | 700 (bold) | 1.2 |
| Display — VS badge | 24px (text-2xl) | 700 (bold) | 1.0 |

Notes:

- VS badge uses bold at 24px (text-2xl) to deliver the dramatic "fighting game" feel specified in CONTEXT.md D-04.
- Winner name on celebration screen: 24px bold (text-2xl font-bold) matching existing result announcement patterns.
- Entry names in match cards: 16px regular — readable in a boxed card context.
- Muted counters ("8 entrants", "Round 2 of 3"): 14px regular text-muted-foreground.

Source: index.css existing type patterns + CONTEXT.md D-04 drama requirement.

## Layout Contract

### Phase: Entry

```
┌─────────────────────────────────────┐
│  [Tutorial Button]                  │
│  ┌───────────────────────────────┐  │
│  │  Options (Label)              │  │
│  │  [Textarea — bulk entry]      │  │
│  │  [Input + Add button]         │  │
│  │  N options entered            │  │
│  └───────────────────────────────┘  │
│  Mode: [Random] [Judge]             │
│  [Start Tournament]                 │
└─────────────────────────────────────┘
```

- Max width: max-w-3xl (768px) — consistent with cards/teams tab containers
- Mode toggle: two `Button` components (variant="default" for active, variant="outline" for inactive), placed above Start button
- Entry card: full width within container, CardContent p-4 space-y-3
- Textarea rows: 6 lines
- Entry count: text-xs text-muted-foreground below textarea ("N options")
- Minimum to start: 2 entries (Start button disabled if < 2)

### Phase: Playing — Bracket Display

```
overflow-x-auto (scrollbar-none on mobile)
┌──────────────────────────────────────────────────────────┐

│ │Option B│  ├──│─│        │  │  │ │        │     │       │
│ └────────┘  │  │ │        │  │  │ └────────┘     │       │
│ ┌────────┐  │  │ └────────┘  │  │                │       │
│ │Option C│──┘  │             │  │                │       │
│ │Option D│     │ ┌────────┐  ├──│                │       │
│ └────────┘     │ │        │──┘  │                │       │

│                │ └────────┘     │                │       │
```

- Bracket container: `flex flex-row gap-8 overflow-x-auto scrollbar-none p-4`
- Each round column: `flex flex-col justify-around gap-4 flex-shrink-0 w-40` (160px fixed width)
- Progressive reveal: column renders only when preceding round is fully resolved
- Connector lines: CSS-only, right border + pseudo-element bridge (no SVG)
- Active matchup (currently animating): border-2 border-bracket-accent on the match card
- Resolved matchups: winner row has text-bracket-accent font-bold; loser row has text-muted-foreground line-through

## Match Card Anatomy

Each `BracketMatch` card contains:

```
┌──────────────────────┐
│ [Option A name]      │  ← bracket-option row, 44px min-height
│──────────────────────│  ← divider (1px border-border)
│ [VS badge] (overlay) │  ← absolute-positioned, center-aligned, text-2xl font-bold
│──────────────────────│
│ [Option B name]      │  ← bracket-option row, 44px min-height
└──────────────────────┘
```

- Card: `Card` component, w-40 (160px), border-2
- Option rows: `px-3 py-2 flex items-center min-h-[44px] text-sm font-normal`
- VS badge: `absolute inset-0 flex items-center justify-center pointer-events-none z-10`
  - Badge text: "VS" — text-2xl (24px) font-bold text-foreground
  - VS badge hidden at rest; `.is-pulsing` class triggers pulse keyframe
- Judge mode option rows: `cursor-pointer hover:bg-muted/50` — pointer feedback on hover, `pointer-events-none` during animation
- Bye matchup card: option rows show "BYE" in text-muted-foreground italic, no VS badge, no animation, border-dashed border-border/40

## Animation Contract

### VS Showdown Sequence (per matchup)

Total duration: ~1200ms

| Phase | Duration | What Happens |
|-------|----------|--------------|
| 1. VS pulse | 0–400ms | VS badge fades/scales in (bracket-vs-pulse keyframe) |
| 2. Shake | 0–600ms | Both option rows apply bracket-shake animation simultaneously |
| 3. Winner flash | 600–1100ms | Winner row plays bracket-winner-flash (amber tint + slide right) |
| 4. Resolve | 1100ms | Winner row: text-bracket-accent + font-bold; Loser: muted + line-through |

CSS constants (in index.css, matching RESEARCH.md Pattern 3):
```
bracket-shake: 0.6s cubic-bezier(0.36, 0.07, 0.19, 0.97)
bracket-vs-pulse: 0.4s cubic-bezier(0.34, 1.56, 0.64, 1) forwards
bracket-winner-flash: 0.5s ease-out forwards
```

ANIMATION_DURATION constant in use-bracket.ts: `1200` (ms).

### Judge Mode Interaction Lock

- During animation (`bracketState.animating === true`): option rows have `pointer-events-none` + `cursor-default`
- User cannot pick winner until VS sequence completes — prevents double-resolution (Pitfall 6 from RESEARCH.md)

### Bye Auto-Advance

- Bye matchups do NOT play the VS animation
- They render immediately with "BYE" label and winner pre-set
- They use a dashed card border to visually distinguish from real matchups
- Round 2 slots fed by byes appear pre-populated when playing phase starts

### Confetti Burst (winner celebration)

- 20 particles
- Each particle: 8px square, border-radius 2px
- Animation: `confetti-fall` 1.2s ease-in forwards (existing keyframe in index.css)
- Stagger: random `animation-delay` 0–0.6s per particle
- Position: random `left` 0–100% within a relative container above the winner card
- Colors cycle through: `["#ffd700", "#ff6b6b", "#4ecdc4", "#45b7d1", "#96ceb4"]`

Source: RESEARCH.md Pattern 6 + existing `.confetti-particle` / `confetti-fall` in index.css.

## Mode Toggle Contract

The Random/Judge mode toggle sits between the entry card and the Start button.

| State | Visual |
|-------|--------|
| Random active | Button variant="default" (filled primary) |
| Judge active | Button variant="default" (filled primary) |
| Inactive mode | Button variant="outline" |
| Either mode during playing phase | Both buttons disabled (mode cannot change mid-tournament) |

Mode toggle disabled during playing and complete phases — mode is locked once tournament starts.
